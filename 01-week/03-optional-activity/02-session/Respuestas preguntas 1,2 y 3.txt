1. ¿Qué pasaría si no definimos correctamente las claves de cada entidad desde el inicio?
Respuesta: El sistema podría enfrentar varios problemas como por ejemplo, podríamos tener registros duplicados, como estudiantes con nombres similares o asignaturas sin relacionarse, lo que dificulta la integridad de los datos, puede generar confusión al intentar acceder o actualizar información y hasta la perdida del acceso completo.

Ejemplo:

Supongamos dos registros de estudiantes en una base de datos académica:

CÓDIGO ESTUDIANTE	NOMBRE ESTUDIANTE	ASIGNATURA	NOTA
123	María López	BD-101	4.5
123	María López	BD-101	4.0

Acá si no definimos una clave única, en este caso, CÓDIGO ESTUDIANTE, podemos tener registros duplicados o inconsistentes, con variaciones en el nombre o notas diferentes para la misma persona.

2. ¿Qué ventajas aporta tener identificadas las entidades antes de pasar al diagrama ER formal?
Respuesta: Facilitaría la comprensión y estructuración de la base de datos, permite saber qué datos debemos almacenar y cómo se interrelacionan, lo que simplifica la creación de relaciones entre las entidades, podemos evitar redundancias y asegurar que no falten datos importantes.

Ejemplo gráfico:

Antes de crear el diagrama ER formal, se tiene que definir las entidades y cómo se relacionan. Por ejemplo, identificando que Estudiante se relaciona con Matrícula y Grupo, mientras que Grupo se relaciona con Asignatura. Lo que ayudaría a evitar redundancias y errores en la estructuración del modelo.

3. ¿Qué información adicional consideras crítica en un sistema académico que aún no hemos modelado?
Respuesta: Podría ser el historial de calificaciones detallado para cada estudiante, que incluya notas finales, evaluaciones, parciales, quices o trabajos. También podría ser el estado académico de los estudiantes, si están activos, en prueba o graduados, y los prerrequisitos para cada asignatura. El registro de asistencia o de comentarios del docente en los trabajos o en cada estudiante.

Una representación gráfica de un modelo más completo puede ayudar a visualizar cómo podrían integrarse elementos adicionales al sistema, como los que había nombrado anteriormente.
